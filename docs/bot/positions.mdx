## Position Details

Tap any position to view detailed information:

### Basic Info

- **Pool** - Trading pair (e.g., SUI/DBUSDC)
- **Maker Address** - Who you're copying
- **Your Position Size** - Total SUI allocated
- **Copy Ratio** - Your sizing strategy (e.g., 10%)
- **Creation Time** - When position started
- **Order Count** - Number of active orders

### DeepBook Details

- **Pool Address** - On-chain pool identifier
- **Balance Manager** - Your balance manager object
- **Capability** - Permission object used
- **Pool State** - Open/Closed status

### Order Information

For each order in the position:

- **Side** - BID (buy) or ASK (sell)
- **Price** - Limit price
- **Quantity** - Order size
- **Status** - Open/Filled/Canceled
- **Transaction Hash** - On-chain proof

<Info>
  Order details are fetched from DeepBook V3 in real-time. May take a few
  seconds to load.
</Info>

## Understanding Position Lifecycle

### 1. Creation Phase

When maker opens a position:

- Bot detects transaction
- Creates position record
- Status: **Monitoring** üîç
- Order count: 0

### 2. Order Placement

When maker places their first order:

- Bot places your corresponding order
- Status changes to: **Active** ‚úÖ
- Order count increments
- You receive notification

### 3. Active Monitoring

While position is active:

- Bot tracks maker's orders
- Syncs your order count
- Updates status in real-time
- Notifies of any changes

### 4. Order Updates

When maker adds more orders:

- Your order count increases
- New orders placed automatically
- Position remains active
- Total exposure increases

### 5. Position Closure

Positions can close when:

- You manually close via bot
- Maker closes their position
- Maker becomes inactive
- You remove the maker

## Managing Positions

### View Position Details

1. Go to `/positions`

### Close a Position

To manually close and stop copying:

1. Select the position
2. Tap **üõë Close Position**
3. Confirm closure
4. Orders are canceled on DeepBook V3
5. Position marked as closed

<Warning>
  Closing a position cancels all open orders but doesn't refund gas or fees
  already spent.
</Warning>

### View on Explorer

To see on-chain data:

1. Select a position
2. Tap **üîç View on Explorer**
3. Opens Sui Explorer in browser
4. View all transactions and orders

### Refresh Position

To update order count manually:

1. Select a position
2. Tap **‚Üª Refresh**

4. Updated data displayed

## Position Limits

### Maximum Active Positions

- **Per maker:** Unlimited
- **Per user:** 50 positions
- **Recommended:** 5-10 active positions

<Tip>More positions = more complexity. Focus on quality over quantity!</Tip>

### Why Limit Positions?

Managing too many positions can lead to:

- Difficulty tracking performance
- High gas fee accumulation
- Capital spread too thin
- Notification overload

## Position Safety

### What Happens If...

<AccordionGroup>
  <Accordion title="Bot goes offline?">
    - Existing positions remain on DeepBook V3
    - Orders stay active on-chain
    - You retain full control via blockchain
    - When bot returns, syncs position state
    - No data is lost
  </Accordion>

<Accordion title="I run out of SUI?">
  - New orders cannot be placed - Existing orders remain active - Position
  marked with error status - You receive low balance notification - Refill SUI
  to resume
</Accordion>

<Accordion title="I run out of DEEP?">
  - Similar to SUI depletion - Cannot pay DeepBook trading fees - Existing
  orders unaffected - Get DEEP to continue copying
</Accordion>

<Accordion title="Maker closes their position?">
  - Your position remains active - Miru doesn't auto-close (yet) - You should
  manually close via `/positions` - Future: Auto-close feature planned
</Accordion>

  <Accordion title="I revoke capability?">
    - Bot cannot place new orders
    - Existing orders stay on DeepBook
    - Positions marked with error
    - Re-grant capability to resume
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Position shows 0 orders but maker is active">
    **Causes:**
    - Bot just created position, maker hasn't placed order yet
    - Maker placed order but it failed
    - Detection lag (usually < 1 second)
    
    **Solution:** Wait a few seconds and refresh. If persists, check maker's address is correct.
  </Accordion>

<Accordion title="Order count not updating">
  **Causes:** - Maker hasn't placed new orders - Network delay in fetching
  DeepBook data - Position is inactive/paused **Solution:** Tap refresh icon.
  Check maker's recent transactions on Explorer.
</Accordion>

  <Accordion title="Can't close position">
    **Causes:**
    - zkLogin session expired
    - Insufficient SUI for cancellation gas
    - Network connectivity issue
    
    **Solution:** Re-authenticate zkLogin, ensure > 0.1 SUI balance, retry.
  </Accordion>

  <Accordion title="Position stuck in monitoring">
    **Possible reasons:**
    - Maker created position but hasn't placed orders
    - Maker position is on different pool
    - Bot missed initial order (rare)
    
    **Solution:** Verify maker's pool address matches. Check maker's transactions. Contact support if issue persists.
  </Accordion>
</AccordionGroup>

## Best Practices

### Daily Routine

1. **Check positions** - Open `/positions` once per day
2. **Review notifications** - Read all position updates
3. **Monitor balances** - Ensure adequate SUI + DEEP
4. **Assess performance** - Are positions profitable?
5. **Adjust strategy** - Close underperformers, add to winners

### Weekly Review

1. **Position audit** - Close old/inactive positions
2. **Maker evaluation** - Are they still performing?
3. **Capital rebalancing** - Adjust position sizes
4. **Fee analysis** - Are gas costs reasonable?
5. **Strategy refinement** - What worked, what didn't?

### Red Flags

Close or pause positions if:

- ‚ùå Maker stops trading for 3+ days
- ‚ùå Multiple failed orders in a row
- ‚ùå Position losing money consistently
- ‚ùå Maker's strategy changed dramatically
- ‚ùå Unusual order patterns
