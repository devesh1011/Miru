# Security Guide

Comprehensive security practices for safe copy trading on Miru.

## Threat Model

Understanding potential risks:

| Threat                        | Impact                         | Likelihood | Mitigation                             |
| ----------------------------- | ------------------------------ | ---------- | -------------------------------------- |
| **Google account compromise** | Complete wallet access         | Low        | Enable 2FA, strong password            |
| **zkLogin session leak**      | Temporary signing access       | Medium     | Secure redirect URLs, expire sessions  |
| **Capability abuse**          | Bot places unauthorized orders | Low        | Monitor positions, revoke capabilities |
| **Malicious maker**           | Copy bad trades                | Medium     | Research makers, diversify             |
| **Smart contract bug**        | DeepBook exploit               | Very Low   | Testnet only, audits                   |
| **Phishing**                  | Reveal sensitive data          | Medium     | Verify sources, no DMs                 |

<Warning>
  Miru is **testnet-only**. Never use real funds until official mainnet launch!
</Warning>

## Wallet Security

### Google Account Protection

Your Google account **is** your wallet. Secure it:

#### Enable Two-Factor Authentication (2FA)

1. Go to [Google Account Security](https://myaccount.google.com/security)
2. Enable **2-Step Verification**
3. Use authenticator app (Google Authenticator, Authy)
4. Add backup codes
5. Enable **Advanced Protection Program** (optional, high security)

<Tip>
  **SMS 2FA is better than nothing**, but authenticator apps are more secure.
</Tip>

#### Strong Password

```
‚ùå Bad: password123
‚ùå Bad: YourName2024
‚úÖ Good: T7$mK9#pL3^xR2@vN5
```

**Requirements:**

- 16+ characters
- Mix of upper/lower/numbers/symbols
- No dictionary words
- No personal info
- Unique (never reused)

Use a **password manager**:

- 1Password
- Bitwarden
- LastPass

#### Dedicated Google Account

Create a **separate Google account** for blockchain:

```
Personal: john.doe@gmail.com (daily use)
Blockchain: johndoe.web3@gmail.com (Miru only)
```

**Benefits:**

- ‚úÖ Isolates crypto activity
- ‚úÖ Limits damage if compromised
- ‚úÖ Better privacy

### zkLogin Session Safety

#### Protect Redirect URLs

After OAuth, Google redirects with sensitive data:

```
https://authflow.sui.io/#id_token=eyJhbGc...
                                  ‚Üë Your JWT token
```

**DO:**

- ‚úÖ Copy URL from browser address bar
- ‚úÖ Paste directly in Miru bot
- ‚úÖ Use external browser (not in-app)
- ‚úÖ Delete the message after successful login

**DON'T:**

- ‚ùå Share URL with anyone
- ‚ùå Post in public channels
- ‚ùå Save in unencrypted notes
- ‚ùå Use public/shared computers

<Warning>
  Anyone with your redirect URL can authenticate zkLogin for a short time. Treat
  it like a password!
</Warning>

#### Session Expiration

zkLogin sessions last **7 days max**:

```
Created: Feb 1, 2026 10:00 AM
Expires: Feb 8, 2026 10:00 AM

After expiration:
- Bot cannot sign transactions
- Positions remain on-chain
- Simply re-authenticate to continue
```

**Best practice:**

- ‚è∞ Renew session every 6 days
- üîî Enable expiration reminders in `/settings`
- üö´ Don't let sessions linger unused

#### Manual Session Revocation

If you suspect session compromise:

1. Open `/wallet`
2. Tap **üîí Revoke Session**
3. Confirm revocation
4. Bot immediately loses signing access
5. Re-authenticate when safe

**When to revoke:**

- Lost your phone
- Shared redirect URL by mistake
- Suspicious activity in positions
- Switching devices

### Private Key Management

#### Export Wallet (Advanced)

Miru lets you export your ephemeral private key:

1. Go to `/wallet`
2. Tap **üîë Export Wallet**
3. Receive private key

**Security:**

- üîí Store in password manager
- üîí Never share with anyone
- üîí Don't save in plain text
- üîí Consider hardware wallet for large amounts

#### Backup Strategies

**Option 1: Password Manager**

```
Entry: Miru Wallet
Username: 0x1a2b3c4d...
Password: [private key]
Notes: Testnet only, Feb 2026
```

**Option 2: Encrypted File**

```bash
# Create encrypted backup
echo "private_key_here" | gpg --encrypt > wallet.gpg

# Decrypt when needed
gpg --decrypt wallet.gpg
```

**Option 3: Paper Backup**

- Write private key on paper
- Store in safe
- Fireproof/waterproof bag
- Tell trusted person where it is

<Note>
  For testnet, simple password manager storage is sufficient. Mainnet requires
  stronger measures.
</Note>

## Capability Security

### Understanding Capabilities

Capabilities grant **limited permissions**:

```
Capability Object: 0xCDEF...
Granted to: 0x9a8b... (bot)

Can:
‚úÖ Place orders on your behalf
‚úÖ Use your balance manager
‚úÖ Interact with DeepBook

Cannot:
‚ùå Withdraw tokens from balance manager
‚ùå Transfer your assets
‚ùå Grant further capabilities
‚ùå Access non-DeepBook protocols
```

### Granting Capabilities

When first activating copy trading:

1. **Review permissions carefully:**

   ```
   Grant capability to:
   Miru Bot (0x9a8b...)

   Permissions:
   - Place limit orders
   - Access balance manager for trading

   Not granted:
   - Withdraw funds
   - Transfer assets
   ```

2. **Verify bot address** (0x9a8b...) matches [official docs](https://docs.miru.xyz)
3. **Approve transaction** via zkLogin
4. **Capability created** and stored

### Monitoring Capabilities

Check active capabilities:

```
/wallet > Manage Capabilities

Active Capabilities (1):
1. DeepBook Trading
   Granted to: 0x9a8b...
   Created: Feb 1, 2026
   Used by: 3 positions

   Actions:
   üîç View Details
   üîí Revoke
```

**Red flags:**

- Multiple capabilities (should only be 1)
- Unknown recipient address
- Capabilities you don't remember granting

### Revoking Capabilities

When to revoke:

- ‚úã Stopping copy trading permanently
- üö® Suspicious position activity
- üîÑ Resetting bot configuration
- üì± Lost device or session

**Process:**

1. `/wallet` > **Manage Capabilities**
2. Select capability
3. **Revoke**
4. Confirm transaction
5. Capability destroyed

**Effect:**

- Bot immediately loses trading access
- Existing positions remain on DeepBook
- Must grant new capability to resume

## Position Safety

### Maker Due Diligence

Before copying a maker:

#### Research Their History

1. **Check Sui Explorer:**

   ```
   https://testnet.suivision.xyz/account/0x1a2b...

   Review:
   - Total transactions
   - DeepBook activity frequency
   - Wallet age
   - Token holdings
   ```

2. **Analyze Trade Performance:**
   - Profit/loss ratio
   - Win rate
   - Average position size
   - Risk management

3. **Community Reputation:**
   - Twitter/Telegram mentions
   - Leaderboards (if available)
   - Known scammer lists

#### Red Flags

‚ùå **Avoid makers with:**

- Brand new wallet (< 1 month old)
- Few transactions (< 50)
- Erratic position sizes (1 SUI then 1000 SUI)
- No history outside DeepBook (suspicious)
- Associated with known exploits

‚úÖ **Prefer makers with:**

- Established history (3+ months)
- Consistent activity
- Transparent strategy
- Community endorsements
- Verifiable track record

### Position Sizing Limits

Never over-allocate to copy trading:

```
Total Portfolio: 100 SUI

‚ùå Risky Allocation:
- Copy trading: 90 SUI (90%)
- Reserve: 10 SUI (10%)

‚úÖ Balanced Allocation:
- Copy trading: 30 SUI (30%)
- Manual trading: 40 SUI (40%)
- Reserve: 30 SUI (30%)
```

**Per-Maker Limits:**

- Conservative: 5-10% per maker
- Moderate: 10-20% per maker
- Aggressive: 20-30% per maker
- Max: 50% per maker (risky!)

### Diversification

Don't put all capital with one maker:

```
‚ùå Single Maker:
- Maker A: 30 SUI (100%)

‚úÖ Diversified:
- Maker A: 10 SUI (33%)
- Maker B: 10 SUI (33%)
- Maker C: 10 SUI (33%)
```

**Benefits:**

- Reduces impact of one bad maker
- Different strategies = lower correlation
- Better risk-adjusted returns
- Learn from multiple approaches

### Phishing Protection

Common phishing tactics:

#### Fake Bot DMs

```
Scammer: "Miru Support"
Message: "Your wallet has an issue. Send your private key to resolve."

Reality: Miru will NEVER DM you first!
```

**Verification:**

- ‚úÖ Official bot: @miru_bot (check username)
- ‚úÖ Never asks for private keys
- ‚úÖ Never asks for redirect URLs
- ‚úÖ All help is in-bot or on docs.miru.xyz

#### Fake Support Channels

```
Scammer creates: "Miru Official Support" (Telegram group)

Reality: Only trust official links from docs!
```

**Verify:**

- Official community: [t.me/miru_community](https://t.me/miru_community)
- Check footer of docs.miru.xyz
- Admins have verified badges

#### Malicious Links

```
Scammer: "Claim your airdrop: mir0.xyz"
                                   ‚Üë zero instead of 'u'

Legit: docs.miru.xyz
```

**Always:**

- Check domain spelling
- Hover before clicking
- Bookmark official sites
- Use docs for all links

## Incident Response

### If Wallet Compromised

**Immediate actions:**

1. **Revoke zkLogin session:**

   ```
   /wallet > Revoke Session
   ```

2. **Revoke all capabilities:**

   ```
   /wallet > Manage Capabilities > Revoke All
   ```

3. **Close all positions (if possible):**

   ```
   /positions > Close All
   ```

4. **Change Google password**

5. **Enable Google 2FA** (if not already)

6. **Contact support:**
   - Report to Telegram: @miru_support
   - Email: security@miru.xyz
   - Include: wallet address, timestamp, details

### If Suspicious Activity Detected

**Example:**

```
Notification: "Position Created: SUI/UNKNOWN"
You: "I didn't authorize this!"
```

**Steps:**

1. **Pause immediately:**
   - Revoke capability
   - Don't interact with suspicious positions

2. **Investigate:**
   - Check maker address
   - Review transaction on Explorer
   - Verify it wasn't you (maybe you forgot?)

3. **Document:**
   - Screenshot everything
   - Note timestamps
   - Save transaction hashes

4. **Report:**
   - Contact support
   - Provide documentation
   - Follow their guidance

### If Lost Access

**Scenarios:**

#### Lost Phone

- ‚úÖ zkLogin still works (Google account accessible from any device)
- ‚úÖ Sessions persist
- ‚úÖ Download Telegram on new device
- ‚úÖ Log in with same account

#### Forgot Google Password

- ‚úÖ Use Google account recovery
- ‚úÖ Recovery email or phone
- ‚úÖ Once recovered, access wallet via zkLogin

#### Lost zkLogin Session (Expired)

- ‚úÖ Re-authenticate via Google
- ‚úÖ Same wallet address restored
- ‚úÖ All positions intact
